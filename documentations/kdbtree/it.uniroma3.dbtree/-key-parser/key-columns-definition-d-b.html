keyColumnsDefinitionDB

abstract fun keyColumnsDefinitionDB(): List<Pair<String, String>> (source)

Returns the key scheme for database table's definition.

This method provides the column scheme for the key type (K) managed by the current KeyParser object. This method will be used by the Connector class at the moment of defining the table persisting the DB-tree. The recommended column names are 'min' and 'max' for simple key representation and 'minx', 'miny', 'maxx', 'maxy' for composite keys. However, regardless of column names, the most important thing is sticking coherent to them for all user-implemented classes (KeyParser and Connector).

Return
the list of pairs: <columnName, columnDefinition>. ColumnDefinition must be understood and interpreted by the underlying database to define the the corresponding column.
